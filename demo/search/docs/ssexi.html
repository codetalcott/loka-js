ssexi.js

ssexi.js extends fixi with Server-Sent Events support, enabling long-lived HTTP streams where servers push HTML to browsers. ssexi adds no new attributes of its own; it integrates with fixi's existing request system and treats responses with Content-Type: text/event-stream as live streams.

How it works with fixi

Rather than introducing new markup, ssexi leverages fixi's established attributes. Incoming SSE messages are swapped into elements using the standard fx-target and fx-swap properties. Existing fixi functionality — modifiers, lifecycle events, transitions — applies without learning new syntax.

Lifecycle events

  • fx:sse:open — stream detected and starting
  • fx:sse:message — fires before each swap
  • fx:sse:swapped — after insertion, useful for auto-scroll
  • fx:sse:close — stream ended normally
  • fx:sse:error — error during streaming

Custom event names (non-JSON SSE events) trigger fx:sse:<eventName> events.

Example

A live-updating feed that appends each message:

<div fx-trigger="fx:init"
     fx-action="/feed"
     fx-swap="beforeend"></div>

The server responds with text/event-stream, emitting data: messages that append to the div as they arrive.
