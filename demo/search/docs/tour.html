A tour through the libraries

The five libraries in the Fixi Project are independent but designed to work together.

fixi.js
Allows an element to issue an HTTP request based on any event and place the response HTML anywhere in the DOM.
moxi.js
Enables scripting for arbitrary element events with helpers for common tasks and DOM-based reactivity.
paxi.js
Provides morphing functionality to merge new content into the DOM while preserving focus and input values.
ssexi.js
Extends fixi to handle streaming Server-Sent Events for live server content.
rexi.js
Offers ergonomic fetch() wrappers to simplify HTTP requests from scripts.

How they fit together

A typical fixi-flavored application uses:

  • plain links for navigation,
  • fixi for server-dependent interactivity,
  • moxi for client-side behaviors,
  • ssexi for live notifications,
  • paxi for stable morphing swaps,
  • rexi for scripted requests.

The combined footprint of all five is ~4.5 KB minified and compressed. The project's philosophy is "less is more."
