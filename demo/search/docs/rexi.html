rexi.js

rexi.js is a fluent wrapper around the browser's fetch() API for making JSON API calls from JavaScript. Unlike fixi (which handles server-driven HTML swaps via attributes), rexi targets scripting contexts with a chainable, promise-based interface.

API

Six HTTP-verb shortcuts: get(), head(), post(), put(), patch(), and del() (delete is a reserved keyword). Each follows verb(url, body?, opts?) with intelligent body normalization for FormData, HTML form elements, plain objects (sent as JSON), strings, and other standard types.

Differences from native fetch()

  • Decorated promises with chainable parsers — .json(), .text(), .html(), .blob() — eliminating the double-await dance.
  • Non-2xx responses throw errors carrying .status and .response.
  • Automatic body normalization based on input type.
  • Built-in lifecycle events rexi:before / rexi:after for global request interception.

Example

let user = await get("/api/me").json()
await post("/api/login", document.forms.login)
await post("/api/users", { name: "Ada" })

Form submission auto-detects the form element and serializes appropriately. Plain objects are JSON-encoded by default.
