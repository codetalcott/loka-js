ssexi — Server-Sent Events sobre fixi

ssexi.js extiende fixi con soporte para HTTP de larga duración: cuando el servidor responde con Content-Type: text/event-stream, los mensajes data: se insertan progresivamente en el objetivo (fx-objetivo / fx-intercambio de fixi).

ssexi no añade ningún atributo nuevo; se monta sobre los de fixi. Lo que sí localiza loka son los nombres de los eventos que ssexi dispara: el orquestador escucha el evento canónico y re-emite su alias en español en el mismo objetivo.

Ejemplo: simular un mensaje SSE

En esta demo no hay un servidor SSE real; disparamos manualmente el evento canónico fx:sse:message con el botón. El listener está registrado con el nombre en español (fx:sse:mensaje) — el orquestador hace el puente.

Español

document.querySelector('#stream')
  .addEventListener('fx:sse:mensaje', (e) => {
    let row = document.createElement('div')
    row.textContent = e.detail.message.data
    e.target.appendChild(row)
  })
